# CraftHelper

Десктопное приложение для расчёта крафтов в играх. Помогает определить необходимое количество базовых ресурсов для создания предмета или максимальное количество крафтов исходя из имеющихся ингредиентов.

## Основные возможности

- Управление рецептами: добавление, редактирование, удаление.
- Управление списком «легкодоступных» ингредиентов (тех, которые не нужно крафтить).
- Расчёт цепочки крафтов для получения нужного количества предмета.
- Расчёт максимального количества крафтов по наличию конкретного ингредиента.
- Удобный интерфейс на CustomTkinter.

## Требования

- Python 3.7 или выше
- Библиотеки: `customtkinter`, `Pillow` (устанавливаются автоматически из `requirements.txt`)
- SQLite3 (входит в стандартную библиотеку Python)

## Установка и запуск из исходного кода

1. Клонируйте репозиторий:
```bash
git clone https://github.com/yourname/craft-helper.git
cd craft-helper
```

2. Создайте и активируйте виртуальное окружение:
```bash
python -m venv venv
# Linux/macOS:
source venv/bin/activate
# Windows:
venv\Scripts\activate
```

3. Установите зависимости:
```bash
pip install -r requirements.txt
```

4. **Инициализируйте базу данных начальными рецептами**:
```bash
python init_db.py
```
Этот скрипт прочитает файлы `assets/recipes.txt` и `assets/easy_ingredients.txt` и создаст базу данных `~/.craft_helper/craft.db` (в Windows — `%USERPROFILE%\.craft_helper\craft.db`).

## Структура файлов данных

Приложение ожидает найти два текстовых файла в папке `assets/`:

### assets/recipes.txt

Каждая строка описывает один рецепт в формате:

```text
Название предмета (количество) - ингредиент1 количество1, ингредиент2 количество2
```

Пример:

```text
Медвежий тартар (10) - Жирный медвежий стейк 5, Речной лук 1.
Барракуда Мрглгагх (10) - Кровь силы 5, Чернная барракуда 5, Масло из молока мускена 4, Речной лук 1.
Жаркое по-сурамарски (10) - Руночешуйный кои 5, Постная рулька 5, Ком морской соли 2, Крупные оливки 4.
Лосось по-дрогбарски (10) - Лосось Крутогорья 5, Улитка из грота Темного Камня 10, Перец далапеньо 4, Крупные оливки 10.
Штормовой скат под соусом из кои (10) - Руночешуйный кои 5, Штормовой скат 5, Ком морской соли 2, Масло из молока мускена 2.
```

### assets/easy\_ingredients.txt

Список ингредиентов, которые считаются базовыми и не требуют крафта. По одному названию на строку (в нижнем регистре). 

Пример:

```text
крупные оливки
масло из молока мускена
ком морской соли
улитки из грота темного камня
перец далапеньо
```

Вы можете редактировать эти файлы перед инициализацией базы или добавлять (редактировать) рецепты через интерфейс приложения позже.

## Сборка исполняемого файла

### Windows

1. Убедитесь, что виртуальное окружение активно и установлены все зависимости (включая `pyinstaller`).
2. Запустите сборочный скрипт:
	cmd
	```
	build_exe.bat
	```
3. Готовый `.exe` файл появится в папке `dist\CraftHelper.exe`.

**Примечание**: Собранное приложение всё равно требует инициализации базы данных! Пользователь должен один раз запустить `init_db.py` (если распространяете исходники) или вы можете распространять уже заполненную базу данных вместе с `.exe` (поместив файл `craft.db` в папку `%USERPROFILE%\.craft_helper\`).

### Linux / macOS

Используйте скрипт `build_exe.sh`:

```bash
chmod +x build_exe.sh
./build_exe.sh
```

## Примечания

- В Windows пути в `--add-data` используют точку с запятой `;` вместо двоеточия.
- Убедитесь, что у вас установлен Python и добавлен в `PATH`.
- Если вы хотите распространять собранное приложение вместе с уже готовой базой данных, вы можете добавить `craft.db` как дополнительный файл данных (но учтите, что пользовательские данные будут сохраняться в `%USERPROFILE%\.craft_helper\craft.db`, поэтому база в папке приложения не будет использоваться после первого запуска). Лучше оставить инициализацию через `init_db.py`.
